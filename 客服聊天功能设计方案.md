# 客服聊天功能设计方案

## 1. 项目现状分析

### 1.1 现有代码结构

**后端代码：**
- `ChatEntity.java` - 客服聊天实体类，包含id、yonghu_id、chat_issue、issue_time、chat_reply、reply_time、zhuangtai_types、chat_types等字段
- `ChatController.java` - 后端接口，提供列表、详情、保存、修改、删除等操作
- `ChatService.java` - 服务接口
- `ChatServiceImpl.java` - 服务实现类
- `ChatDao.java` - 数据访问接口
- `ChatDao.xml` - SQL映射文件

**前端代码：**
- `CustomerServiceView.vue` - 管理员端客服管理页面，包含搜索、列表、查看、处理等功能，但目前使用模拟数据

### 1.2 现有功能分析

- 后端已实现基本的CRUD操作
- 前端已有管理员端界面模板
- 数据库表结构已设计完成
- 缺少用户端聊天界面
- 缺少实时通信功能
- 缺少消息状态管理

## 2. 功能设计

### 2.1 核心功能

1. **用户端聊天功能**
   - 发起咨询
   - 查看聊天历史
   - 接收客服回复
   - 消息状态显示

2. **管理员端管理功能**
   - 查看所有咨询
   - 处理咨询
   - 回复用户
   - 标记处理状态
   - 搜索和筛选

3. **实时通信**
   - 消息实时推送
   - 在线状态显示
   - 未读消息提醒

### 2.2 数据模型优化

**现有ChatEntity字段：**
- id: 主键
- yonghu_id: 提问用户
- chat_issue: 问题
- issue_time: 问题时间
- chat_reply: 回复
- reply_time: 回复时间
- zhuangtai_types: 状态
- chat_types: 数据类型
- insert_time: 创建时间

**优化建议：**
1. 添加 `last_message_time` 字段，记录最后消息时间
2. 添加 `unread_count` 字段，记录未读消息数
3. 添加 `admin_id` 字段，记录处理客服ID
4. 优化 `zhuangtai_types` 枚举值，明确状态定义

### 2.3 状态管理

**消息状态：**
- 0: 待处理（用户已发送，客服未查看）
- 1: 处理中（客服已查看，未回复）
- 2: 已完成（客服已回复）
- 3: 已关闭（会话结束）

**用户状态：**
- 在线
- 离线

## 3. 技术实现方案

### 3.1 后端实现

1. **API接口优化**
   - 完善现有CRUD接口
   - 添加获取用户聊天列表接口
   - 添加获取聊天详情接口
   - 添加发送消息接口
   - 添加标记已读接口

2. **实时通信**
   - 使用WebSocket实现实时消息推送
   - 集成Spring WebSocket或使用第三方库如Netty

3. **消息处理**
   - 实现消息队列，确保消息可靠传递
   - 添加消息存储和检索功能

### 3.2 前端实现

1. **用户端聊天界面**
   - 聊天窗口组件
   - 消息列表组件
   - 消息输入组件
   - 在线状态显示

2. **管理员端界面优化**
   - 连接实际API
   - 添加实时消息提醒
   - 优化处理流程

3. **WebSocket集成**
   - 前端WebSocket客户端实现
   - 消息实时接收和发送
   - 重连机制

### 3.3 数据库优化

1. **索引优化**
   - 为 `yonghu_id`、`zhuangtai_types` 添加索引
   - 为 `issue_time`、`reply_time` 添加索引

2. **分表策略**
   - 考虑按时间或用户ID分表，提高查询性能

## 12. 总结

通过本方案的实施，将大大提升客户服务质量，提高用户满意度，同时减轻客服人员的工作负担，提升工作效率。
