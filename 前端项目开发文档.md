# 网上宠物店前端项目开发文档

## 一、项目概述

### 1.1 项目简介
网上宠物店系统是一个基于 Vue 3 + Spring Boot 的全栈电商平台，提供宠物用品销售、宠物寄养服务、论坛交流等核心功能。

### 1.2 技术栈
- **前端框架**: Vue 3.3.4 (Composition API)
- **构建工具**: Vite 4.4.9
- **路由管理**: Vue Router 4.6.4
- **HTTP 客户端**: Axios 1.13.6
- **UI 组件库**: Element Plus (推荐安装)
- **状态管理**: Pinia (推荐安装)

### 1.3 后端接口信息
- **后端框架**: Spring Boot 2.2.2.RELEASE
- **接口基础路径**: `/wangshangchongwudian`
- **服务端口**: 8080
- **完整接口地址**: `http://localhost:8080/wangshangchongwudian`

**功能描述**: 用户注册、登录、个人信息管理

**API 接口**:
```javascript
// 用户登录
POST /yonghu/login
{
  "username": "string",
  "password": "string"
}

// 用户注册
POST /yonghu/register
{
  "username": "string",
  "password": "string",
  "yonghu_name": "string",
  "yonghu_phone": "string",
  "yonghu_email": "string"
}

// 获取当前用户信息
GET /yonghu/session

// 更新用户信息
PUT /yonghu/update
{
  "id": "number",
  "yonghu_name": "string",
  "yonghu_phone": "string",
  "yonghu_email": "string",
  "yonghu_photo": "string"
}
```

**数据字段**:
- `username`: 账户
- `password`: 密码
- `yonghu_name`: 用户姓名
- `yonghu_phone`: 联系方式
- `yonghu_id_number`: 身份证号
- `yonghu_photo`: 用户头像
- `sex_types`: 性别 (字典表)
- `yonghu_email`: 电子邮箱
- `new_money`: 余额

- `LoginView.vue` - 登录页
- `RegisterView.vue` - 注册页

## 六、开发规范

### 6.1 命名规范
- **文件名**: 使用 PascalCase (如 `ProductList.vue`)
- **组件名**: 使用 PascalCase (如 `<ProductCard />`)
- **变量名**: 使用 camelCase (如 `productList`)
- **常量名**: 使用 UPPER_SNAKE_CASE (如 `API_BASE_URL`)
- **CSS 类名**: 使用 kebab-case (如 `.product-card`)

### 6.2 代码风格
- 使用 Composition API (Vue 3)
- 使用 ES6+ 语法
- 统一使用单引号
- 语句末尾加分号

### 6.3 Git 提交规范
```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构代码
test: 测试相关
chore: 构建/工具链相关
```

## 七、开发环境配置

### 7.1 环境要求
- Node.js >= 14.18.0
- npm >= 8.0.0

### 7.2 安装依赖
```bash
cd frontend
npm install
```

### 7.3 推荐安装的依赖
```bash
# UI 组件库
npm install element-plus

# 状态管理
npm install pinia

# HTTP 客户端（已安装）
npm install axios

# 图标库
npm install @element-plus/icons-vue

# 日期处理
npm install dayjs
```

### 7.4 启动开发服务器

## 八、API 调用示例

### 8.1 在组件中调用 API
```vue
<script setup>
import { ref, onMounted } from 'vue'
import { productApi } from '@/utils/api'

const productList = ref([])
const loading = ref(false)

// 获取商品列表
const fetchProducts = async () => {
  loading.value = true
  try {
    const res = await productApi.getProductList({
      page: 1,
      limit: 10
    })
    productList.value = res.data.list
  } catch (error) {
    console.error('获取商品失败:', error)
  } finally {
    loading.value = false
  }
}

onMounted(() => {
  fetchProducts()
})
</script>
```

```vue
  }
```

## 九、常见问题

### 9.1 跨域问题
开发环境下需要在 `vite.config.js` 中配置代理：
```javascript
export default defineConfig({
  server: {
    proxy: {
      '/wangshangchongwudian': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
})
```

### 9.2 Token 认证
所有需要认证的接口需要在请求头中携带 token：
```javascript
// request.js 中的请求拦截器
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers['token'] = token
  }
  return config
})
```

### 9.3 图片路径处理
后端返回的图片路径需要拼接完整的 URL：
```javascript
const imageUrl = `${baseUrl}${photoPath}`
// 例如：http://localhost:8080/wangshangchongwudian/upload/xxx.jpg
```
